# Infrastructure as code

- DevOps + Agile => Needs faster techniques to provision infrastructure
  - E.g. create test environments & terminate quickly
- Good for disaster recovery.
- There are many tools to automate the underlying infrastructure

## Azure Deployment Environments (2025)

- **On-Demand Test Environments**: Create temporary environments for testing and validation
- **Benefits**:
  - Consistent environment provisioning
  - Version-controlled infrastructure
  - Automated cleanup after testing
  - Integration with CI/CD pipelines
- **Use Cases**:
  - Feature branch testing
  - Pull request validation
  - Load testing environments
  - Production-like testing
- **Integration**:
  - Azure Pipelines and GitHub Actions
  - Bicep and Terraform support
  - Environment templates and policies
  - Cost management and controls

## IaC Tools Comparison (2025)

| Tool | Language | State Management | Learning Curve | Best For |
|------|----------|------------------|----------------|----------|
| **ARM Templates** | JSON | Implicit | Steep | Azure-native deployments |
| **Bicep** | DSL (Domain Specific) | Implicit | Moderate | Simplified Azure deployments |
| **Terraform** | HCL | Explicit | Moderate | Multi-cloud deployments |
| **Pulumi** | General Purpose | Explicit | Varies | Developers using familiar languages |

## Best Practices (2025)

- **Version Control**: All IaC files should be in source control
- **Modular Design**: Create reusable modules and templates
- **Parameterization**: Use parameters for environment-specific values
- **Testing**: Test infrastructure changes in non-production first
- **Security**: Scan IaC files for security misconfigurations
- ** drift Detection**: Monitor for configuration drift
- **Continuous Validation**: Validate infrastructure as part of CI/CD
